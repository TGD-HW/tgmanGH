Detailed Description

Describes one general module.

Structure size is 1024 bytes. TGMotion only copies VendorID and Product ID to the structure and the PLC must handle all the sync manager and FMMUs values.

◆ bUseLRW

uint32_t GENERAL_DEVICE::bUseLRW

controls sending PDO: lower 16 bits: bit 0: if set to 1, use LRW command for PDO (User_PDO_Output_Size and User_PDO_Input_Size must be the same), otherwise use LRD / LWR pair.
bit 1: if set, the LRW is sent also during safe-op phase bit 2: PDO out data protect (see PDO_Send_Lock below) bit 3: PDO in data protect (see PDO_Recv_Lock below) bit 4: use complete SDO for small (up to 4 bytes) data length upper 16 bits: if set to nonzero, determines cycle time multiplier of PDO send/receive e.g. bit[31..16] = 0 or 1: PDO is sent each cycle time bit[31..16] = 2: PDO is sent each second cycle time, etc.

◆ PLC_Send_Lock_Value

volatile uint32_t GENERAL_DEVICE::PLC_Send_Lock_Value

PLC PDO send index: to prevent data race-condition, PLC can change User_PDO_Output[] array only when PLC_Send_Lock_Value and TGM_Send_Lock_Value are the same. After data update, PLC increments PLC_Send_Lock_Value. Just before TGM sends data to EtherCAT, copies them to temporary buffer and after copy set the TGM_Send_Lock_Value to the PLC_Send_Lock_Value. The protection works only if bit2 is set in bUseLRW variable.

◆ TGM_Recv_Lock_Value

volatile uint32_t GENERAL_DEVICE::TGM_Recv_Lock_Value

PLC PDO recv index: to prevent data race-condition, TGM can change User_PDO_Input[] array only when TGM_Recv_Lock_Value and PLC_Recv_Lock_Value are the same. After data read, TGM increments TGM_Recv_Lock_Value. PLC checks if the TGM_Recv_Lock_Value is different to PLC_Recv_Lock_Value. If so, uses the incoming data and after processing, set the PLC_Recv_Lock_Value to be equal to the TGM_Recv_Lock_Value. The protection works only if bit3 is set in bUseLRW variable.
